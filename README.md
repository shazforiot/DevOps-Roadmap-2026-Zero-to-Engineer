## District 1 — Programming Languages

**Start here.** Python opens 80% of DevOps doors — automation scripts, AWS boto3, Ansible modules, and CI/CD pipelines all use Python.

- Python basics: variables, functions, classes, file I/O
- Scripting: working with APIs, JSON, subprocess, argparse
- Automation: writing scripts that replace manual tasks
- Go: learn later for Kubernetes operators and CLI tools

---

## District 2 — Server Administration (Linux)

Over 90% of servers run Linux. Non-negotiable skill.

- Shell navigation: `cd`, `ls`, `grep`, `find`, `cat`, `tail`
- User and permission management: `chmod`, `chown`, `sudo`
- Process management: `ps`, `top`, `kill`, `systemd`
- Package management: `apt`, `yum`, `dnf`
- SSH: key-based auth, configuration, tunneling

## District 3 — Network & Security

The invisible layer everything depends on.

- TCP/IP: connections, ports, handshakes
- DNS: resolution, record types (A, CNAME, MX), `dig`/`nslookup`
- HTTPS/TLS: certificates, handshake, encryption
- Firewalls: `iptables`, security groups, network policies

---

## District 4 — Servers

- **Nginx** — Reverse proxy, load balancing, SSL termination
- **Redis** — In-memory caching, session storage, pub/sub
- **PostgreSQL** — Relational database management, backups, replication

## District 5 — Infrastructure as Code

### Docker — Containerization

Package applications into portable containers.

- Build images with Dockerfiles
- Run and manage containers
- Use docker-compose for multi-service apps
- Networking, volumes, environment variables

### Kubernetes — Container Orchestration

Manage containers at scale — scheduling, scaling, self-healing.

- Pods, Deployments, Services, Ingress
- `kubectl` commands for daily operations
- ConfigMaps, Secrets, Namespaces
- Horizontal Pod Autoscaling

### Terraform — Infrastructure Provisioning

Define cloud resources in HCL code. `terraform apply` creates everything. `terraform destroy` tears it down.

- HCL syntax: resources, variables, outputs
- State management and remote backends
- Modules for reusable infrastructure patterns

### Ansible — Configuration Management

Automate server configuration without SSH-ing into each machine.

- Playbooks, roles, inventory
- Ansible Vault for secrets encryption
- Idempotent configuration

## DORA Metrics — How Elite Teams Measure Success

| Metric | Elite | Low Performer |
|--------|-------|---------------|
| Deployment Frequency | On demand (multiple/day) | Once every 1–6 months |
| Lead Time for Changes | Less than 1 hour | More than 6 months |
| Change Failure Rate | 0–15% | 46–60% |
| Time to Restore Service | Less than 1 hour | More than 1 week |

Source: [DORA State of DevOps Report](https://cloud.google.com/devops/state-of-devops/)
